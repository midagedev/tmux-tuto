# AGENTS.md

## Translation Policy

- 번역 문구(`en`, `ja`, `zh` 등)는 에이전트가 원문을 직접 읽고 문맥을 이해한 뒤 자연스럽게 작성한다.
- 자동 치환/기계 번역 결과를 그대로 복붙하지 않는다.
- 용어는 제품 문맥에 맞게 일관되게 유지한다. (예: session, pane, prefix, playbook)
- 직역보다 사용자 가독성과 실제 UI 문맥을 우선한다.
- `scripts/*`는 번역 생성기가 아니라 반영/정렬/검증 용도로만 사용한다.
- 번역 수정 후 `npm run i18n:verify`를 실행해 로케일 키 정합성을 확인한다.
- 구현 시 기존 동작을 보존하기 위한 우회적 패치보다, 현재 기준에서 가장 타당한 설계를 우선한다.
- 하위 호환은 필요한 범위에서 명시적으로 다루되, 설계를 왜곡하는 임시 우회 로직은 지양한다.
- PR 머지 이후에는 각 레슨이 실제로 실행 가능한지 레슨 전용 E2E(`npm run test:e2e:lessons`)로 검증한다.
