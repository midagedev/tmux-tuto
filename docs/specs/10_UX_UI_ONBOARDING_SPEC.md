# 스펙: UX/UI 디자인 시스템 + 온보딩

## 1. 목표
- 과장 없이 실용적이지만 첫인상에서 "콘텐츠 품질이 높다"는 신뢰를 준다.
- 실무형 사용자가 즉시 학습 시작할 수 있는 구조를 만든다.
- 성취 공유가 부담스럽지 않고 자연스럽게 SNS 공유로 이어지게 한다.

## 1.1 릴리즈 범위 명시
- v1 실제 학습 범위는 `Track A~C`다.
- 랜딩에서 `Track D~E`는 "곧 추가될 고급 트랙"으로만 노출한다.

## 2. 타깃 사용자 심리 모델
- 화려한 마케팅 카피보다 "실제로 배울 수 있는가"를 먼저 확인한다.
- 시간 대비 효율을 중시해 첫 1~2분 내 학습 가치를 판단한다.
- 공유 욕구는 "과시"보다 "실용적인 성취 인증"에서 나온다.

## 3. 디자인 원칙
1. Practical First
- 시각적 장식보다 과제/진행/결과 정보를 우선 노출한다.

2. Calm Confidence
- 강한 과장 표현 대신 단정한 레이아웃과 높은 가독성으로 신뢰를 만든다.

3. Evidence Over Claims
- "최고" 같은 추상 문구 대신 샘플 미션, 실제 화면, 난이도 구조를 보여준다.

4. Shareable Achievement
- 공유는 선택적 행동으로 설계하고, 성취 결과가 깔끔하게 전달되도록 한다.

## 4. 브랜드 톤/카피 가이드
## 4.1 톤
- 담백하고 구체적
- 실무형 어휘 사용
- 과잉 낙관/과장 금지

## 4.2 카피 규칙
- 금지 표현:
  - "혁신", "게임체인저", "압도적", "인생이 바뀌는"
- 권장 표현:
  - "3분 안에 첫 패인 분할"
  - "실습으로 단축키를 손에 익히기"
  - "실전에서 바로 쓰는 tmux 루틴"

## 5. 시각 시스템
## 5.1 컬러 토큰
- 배경:
  - `--bg-base: #F4F5F7`
  - `--bg-surface: #FFFFFF`
- 텍스트:
  - `--text-primary: #111827`
  - `--text-secondary: #4B5563`
- 강조:
  - `--accent: #0F766E`
  - `--accent-soft: #CCFBF1`
- 상태:
  - `--success: #166534`
  - `--warning: #B45309`
  - `--danger: #B91C1C`

## 5.2 타이포그래피
- 헤드라인: `Space Grotesk`
- 본문(UI): `IBM Plex Sans KR`
- 코드/단축키: `JetBrains Mono`

## 5.3 형태/간격
- radius:
  - 카드 12px
  - 버튼 10px
- 간격 스케일:
  - 4, 8, 12, 16, 24, 32, 48
- 그림자:
  - 얕은 그림자 1단계만 사용 (과도한 depth 금지)

## 6. 페이지 구조 설계
## 6.1 랜딩 페이지 (첫인상 핵심)
- 섹션 순서:
1. Hero
2. "10분 안에 할 수 있는 것" 증거 섹션
3. 실제 연습 화면 미리보기(시뮬레이터 캡처/데모)
4. 커리큘럼 구조 (Track A~C + D/E preview)
5. 실무 플레이북 미리보기 (`권장 tmux.conf`, `세션 유지`, `Tailscale SSH`)
6. 성취 공유 예시 카드
7. CTA: `지금 3분 미션 시작`

### Hero 규칙
- 헤드라인 1개 + 서브카피 1개 + CTA 1~2개
- 숫자 기반 가치 전달:
  - 예: "첫 3분, tmux 패인 분할을 실습으로 익히기"

## 6.2 Learn 페이지
- 좌측: 트랙/챕터 내비
- 중앙: 레슨 설명 + 미션 진입
- 우측: 예상 소요시간, 선행조건, 완료 상태

## 6.3 Practice 페이지
- 좌측: 현재 미션/목표
- 중앙: 터미널 시뮬레이터
- 우측: 힌트/단축키/실제 tmux 차이 안내

## 6.4 Cheatsheet 페이지
- 상단 검색바 고정
- 결과는 카드형 + 즉시 실습 CTA
- 최근 검색/자주 찾는 항목 별도 블록

## 6.5 Progress 페이지
- 상단: XP/레벨/스트릭
- 중단: 트랙별 진행률
- 하단: 다음 추천 실습 + 공유 CTA

## 7. 온보딩 플로우 (5단계)
1. Welcome (20~30초)
- 이 사이트가 해주는 것/안 해주는 것 명확히 안내
- "실제 tmux 실행이 아닌 고충실도 시뮬레이터" 명시

2. 목표 선택 (30초)
- 선택지:
  - "업무용 기초 빨리 익히기"
  - "단축키 자주 까먹어서 복습 중심"
  - "고급 루틴까지 완주"

3. 입력 선호 설정 (20초)
- prefix 선택: `Ctrl+b`(권장), `Ctrl+a`
- 키보드 레이아웃(맥/윈도우) 선택

4. 첫 실습 (2~3분)
- "pane 2개 만들기" 초단기 미션 제공
- 성공 시 즉시 피드백 + 다음 단계 안내

5. 개인화 완료
- 추천 트랙 1개 자동 제시
- 추천 플레이북 1개 자동 제시
- `오늘 1개만 끝내기` CTA 제공

## 8. 첫인상 신뢰를 높이는 콘텐츠 전략
- 랜딩 상단에서 실제 학습 콘텐츠 일부를 그대로 공개:
  - 샘플 미션 1개
  - 힌트 예시 1개
  - 완료 기준(채점 룰) 1개
- 실무 플레이북 샘플 1개(`tmux.conf` 일부 + 검증 체크리스트) 공개
- 학습 깊이를 보여주는 "커리큘럼 미리보기" 공개
- 업데이트/개선 내역(간단 changelog) 제공

## 9. 공유 유도 UX (비과장)
- 공유 CTA는 완료 순간에만 노출한다.
- 기본 카피는 성취 사실 중심으로 구성:
  - "Track A 완료: 세션/윈도우/패인 기본 조작 익힘"
- 공유 전 미리보기 카드 확인 가능
- 공유는 강제하지 않고 "복사 링크"를 기본 액션으로 둔다.

## 10. 공유 카드 디자인 규칙
- 정보 우선순위:
1. 마일스톤 이름
2. 달성 내용 한 줄
3. 레벨/연속 학습일
4. URL
- 카드 스타일:
  - 배경은 차분한 그라데이션(저채도)
  - 단축키 키캡 모티프를 패턴으로 약하게 사용
  - 과도한 뱃지/이펙트 금지

## 11. 모션/인터랙션
- 전환 애니메이션 140~180ms
- 성공 피드백 애니메이션 300ms 이하
- 무한 깜빡임/강한 패럴랙스 금지

## 12. 접근성
- 텍스트 대비 WCAG AA 충족
- 키보드 전용 내비게이션 100% 가능
- 온보딩 단계별 포커스 트랩/스크린리더 라벨 제공

## 13. 반응형 전략
- Desktop 우선 최적화
- Tablet:
  - 우측 패널을 하단 드로어로 이동
- Mobile:
  - Practice는 미니 실습/리뷰 중심
  - 긴 입력 실습은 데스크톱 권장 안내

## 14. 검증 지표 (Cloudflare 라우트 기반)
- 첫인상 유지율:
  - 랜딩 진입 -> `/learn` 혹은 `/practice` 이동 비율
- 온보딩 완료율:
  - `/onboarding/start` -> `/onboarding/done` 도달률
- 첫 실습 성공률:
  - `/onboarding/first-mission/passed` 진입 비율
- 공유 전환율:
  - milestone 달성 라우트 대비 `/share/*` 진입 비율

## 15. 사용성 테스트 시나리오
1. 첫 방문 10초 내 가치 파악 가능 여부
2. 온보딩 3분 내 완료 가능 여부
3. 단축키 검색 후 실습 전환 가능 여부
4. 완료 후 공유 행동이 자연스럽게 발생하는지

## 16. DoD
- 랜딩/핵심 페이지/온보딩 와이어프레임 확정
- 디자인 토큰/컴포넌트 규약 확정
- 공유 카드 템플릿 확정
- 사용성 테스트 체크리스트 통과
